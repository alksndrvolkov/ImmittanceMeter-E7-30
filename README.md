# Библиотека для работы с измерителем иммитанса МНИПИ E7-30(Python)

Удобная Python-библиотека для общения с измерителем иммитанса через последовательный порт (RS-232/USB-UART). Позволяет задавать частоту тестирования, включать смещение (bias) и считывать модуль импеданса и фазу. Подходит для автоматизации измерений, сканирования по частоте и интеграции в лабораторные стенды.

####  !!! Данная библиотека была создана без участия компании изготовителя МНИПИ Е7-30 !!!

Возможности библиотеки:
- Автоопределение времени обновления прибора (скорости измерений)
- Считывание |Z| и φ (градусы)
- Минимум зависимостей (только pyserial)
- Кроссплатформенно: Windows / Linux / macOS

## Подключение

Параметры порта (заданы в коде):

## Быстрый старт

Простейший пример: установить частоту 1100 Гц, смещение 0.15В и прочитать модуль импеданса и фазу.

```python
from E7_30_ImmittanceMeter_V2 import ImmittanceMeter

meter = ImmittanceMeter("COM2", 0.2) # некоторое время будет инициализация
print("Reading rate:", meter.measurements_rate) # вычисление выбранной на приборе скорости измерения
meter.set_frequency(1100) # частота в Гц
meter.set_bias_voltage(0.15) # напряжение смещения в В
impedance_module, impedance_phase = meter.read_impedance()
print("Impedance module: ", impedance_module, " Ω")
print("Impedance phase: ", impedance_phase, " °")
meter.close_serial()
```

## API

Ниже кратко описаны публичные методы класса ImmittanceMeter.

### Инициализация

```python
ImmittanceMeter(COM_PORT: str, COM_timeout: float, frame_timeout: float = 3.0)
```

- COM_PORT — строка порта, например "COM3" (Windows) или "/dev/ttyUSB0" (Linux/macOS).
- COM_timeout — timeout для операций чтения/записи порта (секунды). Значение 0.2 должно подойти.
- frame_timeout — максимальное время ожидания кадра данных при чтении |Z| и φ (по умолчанию 3 секунды).

При инициализации библиотека:
- открывает порт с параметрами 9600 8N1;
- пытается определить выставленную на приборе скорость измерений (measurements_rate: "Fast" или "Middle") методом find_update_time(). На основании скорости выбирается update_time.

### Закрытие порта

```python
close_serial() -> None
```

Корректно закрывает последовательный порт. Рекомендуется вызывать в блоке finally.

### Установка частоты

```python
set_frequency(frequency: int) -> None
```

- Задаёт частоту возбуждения в диапазоне 25…3_000_000 Гц (целое число).
- Отправляет команду и ждёт update_time перед следующим запросом.

Ошибки:
- ValueError — если частота вне допустимого диапазона.

### Установка смещения (bias)

```python
set_bias_voltage(bias_voltage: float) -> None
```

- Устанавливает напряжение смещения 0.00…40.00 В (шаг 0.01 В).
- Значение аккуратно округляется до сотых (ROUND_HALF_UP).
- Отправляет команду и ждёт update_time.

Ошибки/предупреждения:
- ValueError — если напряжение вне диапазона.
- Warning — если значение имеет более двух знаков после запятой (будет округлено).

### Чтение импеданса

```python
read_impedance() -> tuple[float, float] | None
```

- Запрашивает измерение у прибора.
- Возвращает кортеж (z_mag, phi_deg), где:
  - z_mag — модуль импеданса |Z|, Ом (float)
  - phi_deg — фаза, градусы (float)
- Возвращает None, если кадр не получен за frame_timeout.

Примечание: после отправки запроса библиотека ждёт update_time (подобран автоматически при инициализации).

### Дополнительно (служебные методы)

- find_update_time() — подбирает update_time, пробуя быстрый (≈0.3 c) и средний (≈1 c) режимы. При успехе выставляет measurements_rate.
- test_update_time(update_time: float) — проверяет, удаётся ли прочитать ответ при данном времени ожидания.
- parse_frame(frame: bytes) -> tuple[float, float] — разбирает бинарный кадр ответа и возвращает (|Z|, φ°).
- identity() — заглушка (пока не реализовано): предполагается команда опроса идентификатора прибора.

## Формат кадра и протокол

Команды и ответы используют заголовок 0xAA и код команды:

- Установка частоты: [0xAA, 0x43] + 4 байта частоты (BE, unsigned).
- Установка bias: [0xAA, 0x46] + 2 байта напряжения×100 (BE, unsigned).
- Запрос импеданса: [0xAA, 0x48], ожидается ответ с тем же заголовком.

Ответ на запрос импеданса:
- Заголовок: 0xAA 0x48
- Далее 18 байт полезных данных (всего 20 байт вместе с заголовком).
- В текущей реализации парсера два 32-битных float (Big Endian) располагаются по смещениям:
  - |Z| в байтах [12:16]
  - φ (рад) в байтах [16:20]
- Фаза переводится в градусы: φ° = φ(рад) × 180/π

Замечания:
- Порядок байт для чисел — Big Endian (BE).

## Диагностика и устранение неполадок

- Ошибка "Update time not found."
  - Проверьте правильность COM-порта и подключение.
  - Убедитесь, что прибор включён и готов к измерениям.
  - Попробуйте увеличить COM_timeout / frame_timeout.
  - Подключайтесь напрямую к прибору без стороннего ПО, которое может удерживать порт.

- read_impedance() возвращает None
  - Данные не успевают прийти при текущем update_time. Переинициализируйте класс (будет подобрано другое время) или вручную увеличьте задержки.
  - Проверьте кабель/переходник USB-UART, драйверы, права доступа.
  - Убедитесь, что частота в рабочем диапазоне, а измеряемый объект подключён корректно.

- ValueError при set_frequency или set_bias_voltage
  - Значения вне допустимых диапазонов (25…3_000_000 Гц и 0.00…40.00 В соответственно).
